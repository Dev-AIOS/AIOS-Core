## 1. السياق الموحد والافتراضات الحاكمة

هذا القسم هو `الدستور` الذي تُبنى عليه كل الحسابات في المستند. أي رقم يتعارض معه يُعتبر خطأ ويجب تصحيحه.

### 1.1 الافتراضات الأساسية

| البند | القيمة المعتمدة | ملاحظة / مصدر |
|---|---|---|
| سعر الصرف | 1 دولار = 50 جنيه مصري ⚠️ | يُراجع شهرياً؛ حساسية النموذج عالية له |
| ضريبة القيمة المضافة | 14% ⚠️ | تُطبق على الخدمات الرقمية؛ أسعار المستهلك المعروضة شاملة الضريبة |
| رسوم بوابات الدفع | ~2.5–3.5% | Paymob القياسي = 2.75% + 3 ج.م للعملية؛ فودافون كاش = 3–5 ج.م ثابتة |
| هامش إعادة بيع التوكنات | 25–40% | الهامش المستهدف فوق تكلفة المزود قبل رسوم الدفع |
| معدل تحويل Freemium | 1–3% | افتراض محافظ مناسب للسوق المصري |
| ARPU المستهدف | 150–250 ج.م شهرياً | يختلف حسب القناة ومزيج الخطط |
| رأس المال | شبه معدوم | الأولوية القصوى لقنوات دخل سريعة بدون تكلفة بناء عالية |
| متوسط عمر العميل (LTV) | 12–18 شهراً | افتراض أولي، يُحسَّن ببرامج الولاء |

### 1.2 الاستهلاك المرجعي للخطط

| الخطة | السعر الشهري (شامل الضريبة) | الاستهلاك المرجعي | النماذج المسموحة |
|---|---|---|---|
| مجاني | 0 ج.م | ~200 ألف توكن/شهر — حد يومي 10 آلاف | GPT-4o-mini, Gemini Flash, Mistral Tiny |
| توفيري | 75 ج.م | ~600 ألف توكن/شهر | GPT-4o-mini, Gemini Flash, Claude Haiku, Mistral |
| أساسي | 150 ج.م | ~1.5 مليون توكن/شهر | كل النماذج المتوسطة + 20 استدعاء API/يوم |
| محترف | 350 ج.م | ~5 ملايين توكن/شهر | كل النماذج + 100 استدعاء API/يوم + أولوية |
| فرق | 250 ج.م للعضو/شهر | ~3 ملايين توكن لكل عضو + لوحة تحكم | كل النماذج + إدارة الفرق + تقارير |
| API | حسب الرصيد | تسعير لكل مليون توكن | حسب قائمة الأسعار المنفصلة |

### 1.3 منهجية التقدير المالي في هذا المستند

1. كل إيراد متوقع يُعرض بثلاثة سيناريوهات: متشائم (P80) / أساسي (P50) / متفائل (P20).
2. الإيراد المذكور هو الإيراد الإجمالي (Gross Revenue) قبل التكاليف ما لم يُذكر خلاف ذلك، مع عرض الهامش التقريبي عند الحاجة.
3. الأشهر تُحسب من بدء التشغيل الفعلي (إطلاق MVP). الشهر 1 تعني أول شهر بعد الإطلاق العام.
4. الأرقام ليست وعوداً بل أدوات تخطيط؛ قاعدتنا: خطط بالمتشائم، واعمل للأساسي، واحتفل بالمتفائل.
5. كل تقديرات الإيرادات مبنية على نموذج نمو مرجعي واحد لضمان الاتساق بين القنوات.

### 1.4 نموذج النمو المرجعي

| المؤشر | الشهر 6 | الشهر 12 | الشهر 18 | الشهر 24 |
|---|---|---|---|---|
| مستخدمون مسجلون (تراكمي) | 5,000 | 20,000 | 38,000 | 60,000 |
| مشتركون مدفوعون (نشطون) | 75 | 350 | 650 | 1,100 |
| معدل التحويل الفعلي | 1.5% | 1.75% | 1.7% | 1.8% |
| ARPU شهري مخلوط | ~170 ج.م | ~180 ج.م | ~190 ج.م | ~200 ج.م |
| MRR (إيراد شهري متكرر) | ~13 ألف ج.م | ~63 ألف ج.م | ~124 ألف ج.م | ~220 ألف ج.م |

> **تحدي الافتراض:** هل 60,000 مسجل في 24 شهراً واقعي برأس مال تسويق شبه معدوم؟ هذا يفترض نمواً عضوياً قوياً (محتوى + إحالة + مجتمعات). إذا فشل النمو العضوي، يُخفَّض كل شيء بنسبة 50–70%.

### 1.5 تحليل حساسية سعر الصرف

سعر الصرف هو المتغير الأكثر خطورة في النموذج لأنه يحدد تكلفة البضاعة المباعة (COGS) بالدولار بينما الإيرادات بالجنيه.

| سعر الصرف | COGS % من الإيراد (عند هامش 30%) | هامش الربح التشغيلي التقريبي |
|---|---|---|
| 40 ج.م / $ | 23% | 62–65% |
| 50 ج.م / $ (الأساس) | 29% | 55–58% |
| 60 ج.م / $ | 34% | 48–52% |
| 75 ج.م / $ | 43% | 38–42% |
| 100 ج.م / $ | 57% | 22–28% |

> **الاستنتاج:** عند سعر صرف 75+، يتحول النموذج من هامش مرتفع إلى هامش متوسط. الحلول: (أ) تحوط طبيعي بشراء أرصدة بالدولار عند الانخفاض، (ب) رفع الأسعار بما يتناسب مع تغير السعر، (ج) التفاوض مع مزودين يقبلون الدفع بالجنيه عبر وسيط محلي.

### 1.6 تحليل حساسية رسوم بوابات الدفع

| البوابة | رسوم المعاملة | ملاحظات |
|---|---|---|
| Paymob (بطاقات) | 2.75% + 3 ج.م | الأكثر انتشاراً في مصر |
| فودافون كاش | 3–5 ج.م ثابتة | ممتاز للمبالغ الصغيرة (أقل من 200 ج.م) |
| أورانج كاش | 3–5 ج.م ثابتة | تغطية أقل |
| InstaPay | 1–3 ج.م ثابتة | رسوم منخفضة لكن المدفوعات المتكررة محدودة |
| تحويل بنكي | 0–10 ج.م | للشركات والمبالغ الكبيرة |

> **الأثر:** على خطة 150 ج.م عبر Paymob، تبلغ الرسوم 2.75% × 150 + 3 = 7.1 ج.م (4.7%). عبر فودافون كاش، الرسوم 5 ج.م (3.3%). الفرق بسيط لكنه يتراكم عند آلاف المعاملات.

## 2. القناة 1: الاشتراكات Subscriptions

### 2.1 فلسفة التصميم

الاشتراكات هي العمود الفقري للنموذج. في السنة الأولى، ستشكل 70%+ من الإيرادات. لذلك، أي قرار خاطئ في التسعير أو الميزات سيكون مكلفاً.

مبادئ التصميم:
1. **التدرج الواضح:** كل خطة أعلى تُقدم قيمة إضافية واضحة تُبرر فرق السعر.
2. **حاجز الدخول المنخفض:** وجود خطة توفيري 75 ج.م تلتقط الشريحة الواسعة من المستخدمين المصريين ذوي الدخل المحدود.
3. **التحويل التصاعدي:** تصميم الميزات بحيث يشعر المستخدم بالحاجة للترقية (مثلاً: يستهلك الـ 600 ألف توكن بسرعة ويضطر للترقية).
4. **الشمولية:** كل خطة يجب أن تكون قابلة للاستخدام دون شراء إضافي (مع وجود رصيد إضافي اختياري).

**مبرر وجودها:**
- محرك اكتساب المستخدمين (Top of Funnel)

**التكلفة على المنصة:**
- المستخدم المجاني العادي يستهلك ~30% من حده (60 ألف توكن/شهر)
- تكلفة ~3-5 ج.م شهرياً (بهامش 30% من سعر التجزئة)
- عند 60,000 مستخدم مجاني في الشهر 24، التكلفة الشهرية ~180-300 ألف ج.م — يجب إدارة هذا الرقم بدقة

## 3. القناة 2: نظام الرصيد Credits

### 3.1 فلسفة التصميم

نظام الرصيد هو قناة التحصيل الأسرع لأنه:
1. يلتقط من لا يستطيع/لا يريد اشتراكاً
2. يتيح الدفع المرن (ادفع كام ما تستخدمش)
3. يمكن ربطه ببطاقات الهدايا وكروت الشحن
4. قاعدة: 1 رصيد = 1 ج.م مصري (تبسيط تام)

### 3.2 باقات الرصيد

| الباقة | السعر | الرصيد | المكافأة (Bonus) | الرصيد الإجمالي | السعر الفعلي للرصيد |
|---|---|---|---|---|---|
| صغيرة | 50 ج.م | 50 رصيد | 0 | 50 | 1.00 ج.م للرصيد |
| متوسطة | 100 ج.م | 100 رصيد | 5 رصيد (5%) | 105 | 0.95 ج.م للرصيد |
| كبيرة | 250 ج.م | 250 رصيد | 25 رصيد (10%) | 275 | 0.91 ج.م للرصيد |
| ضخمة | 500 ج.م | 500 رصيد | 75 رصيد (15%) | 575 | 0.87 ج.م للرصيد |

**مبرر المكافآت:**
- نظام المكافآت يرفع متوسط قيمة الشراء (AOV)
- عند شراء باقة 500 ج.م، المستخدم يحصل على 575 رصيد (توفير 13%)
- هذا يُشجّع على شراء الباقات الأكبر، مما يُحسّن التدفق النقدي

| 4 | المستخدم يرى رصيده المتبقي وتاريخ انتهاء الصلاحية في لوحة التحكم |

## 9. القناة 8: سوق خوادم MCP (الناشئ)

### 9.1 ما هو MCP؟

Model Context Protocol (MCP) هو بروتوكول ناشئ من Anthropic يتيح للـ AI Agents الاتصال بخوادم خارجية (قواعد بيانات، APIs، خدمات) لجلب البيانات وتنفيذ الإجراءات. هذا السوق في بدايته (2025-2026) وهناك فرصة كبيرة للريادة في العالم العربي.

### 9.2 المنتجات المعروضة في سوق MCP

| نوع الخادم | الوصف | السعر الشهري المتوقع |
|---|---|---|
| MCP Server لقاعدة بيانات | الاتصال بقواعد SQL/NoSQL عبر الـ Agent | 100-300 ج.م |
| MCP Server لأداة خارجية | تكامل مع Google Sheets, Notion, Slack, إلخ | 50-200 ج.م |
| MCP Server لـ API مالي | جلب أسعار الأسهم، العملات، الذهب | 150-500 ج.م |
| MCP Server لخدمة حكومية | الاستعلام عن الخدمات الحكومية المصرية | 100-400 ج.م |
| MCP Server للطقس | بيانات الطقس والتوقعات | 30-100 ج.م |
| MCP Server ترجمة | ترجمة فورية متخصصة بالمجال الطبي/القانوني | 100-300 ج.م |
| MCP Server بحث | بحث متقدم في قواعد بيانات متخصصة | 200-500 ج.م |

### 9.3 نموذج العمل

| العنصر | القيمة |
|---|---|

| خيار البيع بنظام الترخيص السنوي | 20% عمولة |

| حاضنة MCP (استضافة الخادم) | 200 ج.م/شهر + 10% من الإيراد |

### 9.4 لماذا MCP مهم استراتيجياً؟

1. السوق لا يزال في مراحله الأولى — الرياديون سيبنون moat تنافسية
2. العالم العربي يفتقر تماماً إلى MCP Servers بالعربية
3. كل MCP Server هو حجر بناء في ecosystem المنصة
4. المطورون الذين يبنون MCP Servers سيصبحون جزءاً من مجتمع المنصة
5. MCP هو مستقبل الـ Agentic AI — التواجد المبكر يعني موقع قيادي

## 11. القناة 10: الإعلانات — نقاش صريح مع البدائل

### 11.1 الموقف المبدئي

الإعلانات مرفوضة كقناة دخل رئيسية لمنتج مدفوع. إليكم الأسباب:

1. **تضارب المصالح:** المستخدم يدفع اشتراكاً ثم يشاهد إعلانات؟ هذا نموذج متناقض.
2. **تجربة المستخدم:** الإعلانات تُدمّر تجربة المستخدم في منتج يعتمد على التركيز والإنتاجية.

4. **تسرب المستخدمين:** المستخدمون يهاجرون من المنتجات التي تعرض إعلانات إلى المنافسين الخاليين منها.
5. **سمعة العلامة:** منتج AI بإعلانات = منتج رخيص.

### 28.1 أسواق الخليج — فرصة ذهبية
